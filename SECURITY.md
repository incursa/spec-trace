# Security

This repository primarily contains documentation, templates, schemas, and examples.

Most issues reported here will be specification, documentation, or example alignment issues and should be opened as a normal issue or pull request.

## Reporting Security Concerns

If you believe you have found a security-sensitive problem in the repository contents, release process, or supporting automation:

- use GitHub's private security advisory flow if it is enabled for the repository
- otherwise contact the maintainers through the most direct private channel available to you

Do not file sensitive security details in a public issue unless the maintainers ask you to do so.

## What Belongs In Issues Or Pull Requests

Use normal issues or pull requests for:

- unclear wording
- contradictions in the standard
- broken examples
- schema or template mismatches
- suggested improvements to public documentation

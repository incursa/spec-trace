# SpecTrace Spec Suite

This folder contains the canonical SPEC suite for the `spec-trace` standard.

The suite intentionally uses the standard to specify itself. Canonical authored sources are the JSON artifacts in this directory. This index is a navigation aid only.

Canonical entry points:

- [`SPEC-STD.json`](./SPEC-STD.json) - core definitions, authority model, and publication rules
- [`SPEC-ID.json`](./SPEC-ID.json) - identifier policy and grouping-key registry
- [`SPEC-LIN.json`](./SPEC-LIN.json) - trace lineage and requirement evolution
- [`SPEC-PRF.json`](./SPEC-PRF.json) - conformance profiles and enforcement levels
- [`SPEC-RPT.json`](./SPEC-RPT.json) - derived reporting dimensions and attestation views
- [`SPEC-EVD.json`](./SPEC-EVD.json) - generated evidence snapshots and merge semantics
- [`SPEC-LAY.json`](./SPEC-LAY.json) - repository layout and artifact placement
- [`SPEC-TPL.json`](./SPEC-TPL.json) - canonical field model and requirement grammar
- [`SPEC-SCH.json`](./SPEC-SCH.json) - schemas and validation rules
- [`SPEC-EXM.json`](./SPEC-EXM.json) - worked examples and traceability coverage

Root files such as [`README.md`](../../../README.md), [`overview.md`](../../../overview.md), [`layout.md`](../../../layout.md), and the templates summarize this suite but do not override it.
